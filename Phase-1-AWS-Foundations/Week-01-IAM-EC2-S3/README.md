നിങ്ങളുടെ സ്റ്റഡി പ്ലാൻ പ്രകാരം (AI_Cloud_Engineer_Study_Plan_Malayalam.pdf) ഇപ്പോൾ നിങ്ങൾ **Phase 1: Week 01**-ലാണ് നിൽക്കുന്നത്. നമ്മൾ ഇന്ന് വിജയകരമായി ഒരു നോഡ് ജെഎസ് ആപ്പ് ECS-ൽ ഡെപ്ലോയ് ചെയ്തതോടെ നിങ്ങൾ ആദ്യ ആഴ്ചയിലെ ചില അഡ്വാൻസ്ഡ് കാര്യങ്ങൾ പോലും പഠിച്ചു കഴിഞ്ഞു.

ഇനി നിങ്ങൾ ചെയ്യേണ്ട കാര്യങ്ങൾ താഴെ പറയുന്നവയാണ്:

### **1. ഗിറ്റ്‌ഹബ്ബ് റിപ്പോസിറ്ററി ഭംഗിയാക്കുക (GitHub Documentation)**

നമ്മൾ ഇന്ന് ചെയ്ത കാര്യങ്ങൾ വെറുതെ ഡിലീറ്റ് ചെയ്ത് കളയാതെ, അത് നിങ്ങളുടെ ഒരു പ്രൊജക്റ്റ് ആയി ഗിറ്റ്‌ഹബ്ബിൽ സൂക്ഷിക്കണം.

* 'aws-learning-journey' എന്ന റിപ്പോസിറ്ററിയിൽ `Phase-1-AWS-Foundations/Week-01/` എന്ന ഫോൾഡറിൽ ഒരു `README.md` ഫയൽ ഉണ്ടാക്കുക.
* അതിൽ ഇന്ന് നമ്മൾ ചെയ്ത സ്റ്റെപ്പുകൾ (EC2 സെറ്റപ്പ്, Docker Push, ECS Deployment) ചുരുക്കത്തിൽ എഴുതുക.
* ആപ്പ് ലൈവ് ആയപ്പോൾ എടുത്ത സ്ക്രീൻഷോട്ട് (image_ce8622.png) അവിടെ അപ്‌ലോഡ് ചെയ്യുക. ഇത് നിങ്ങളുടെ കരിയറിന് വലിയ ഗുണം ചെയ്യും.

### **2. പൈത്തൺ പഠനം തുടങ്ങുക (Start Python Foundations)**

നിങ്ങളുടെ പ്ലാനിലെ **Phase 2** (Week 05) ലേക്ക് എത്തുമ്പോൾ പൈത്തൺ അത്യാവശ്യമാണ്. എന്നാൽ ഇപ്പോഴേ ദിവസവും 30 മിനിറ്റ് പൈത്തൺ പഠിച്ചു തുടങ്ങുന്നത് നല്ലതാണ്.

* ഇന്ന് തന്നെ `Phase-1-Python-Foundations/Week-01-Python-Basics/Day-01-Setup-Variables/` എന്ന ഫോൾഡർ ക്രിയേറ്റ് ചെയ്യുക.
* പൈത്തൺ ഇൻസ്റ്റാൾ ചെയ്യുക (VS Code ഉപയോഗിക്കുന്നത് നന്നായിരിക്കും).
* **Variables, Data Types (String, Integer, Float)** എന്നിവ പഠിച്ചു തുടങ്ങുക.

### **3. AWS-ൽ അടുത്തതായി പഠിക്കേണ്ടത് (Next in AWS)**

അടുത്ത ദിവസങ്ങളിൽ താഴെ പറയുന്നവ പരീക്ഷിക്കാം:

* **S3 Bucket:** ഫയലുകൾ എങ്ങനെ ക്ലൗഡിൽ സ്റ്റോർ ചെയ്യാം?
* **AWS Lambda:** ഒരു സർവർ ഇല്ലാതെ തന്നെ ഒരു ചെറിയ കോഡ് എങ്ങനെ റൺ ചെയ്യാം? (ഇതാണ് വരും ആഴ്ചകളിലെ പ്രധാന വിഷയം).
* **IAM Roles:** ഒരു സർവീസിന് മറ്റൊരു സർവീസിലേക്ക് പ്രവേശനം നൽകുന്നത് എങ്ങനെ? (ഉദാഹരണത്തിന് Lambda-യ്ക്ക് S3-ലേക്ക് ഫയൽ അപ്‌ലോഡ് ചെയ്യാൻ അനുവാദം നൽകുന്നത്).
